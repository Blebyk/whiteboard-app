# Whiteboard App

Веб-приложение для работы на виртуальной доской, вдохновлённое Miro, FigJam и Excalidraw.

---

## Стек технологий

* **Next.js 15** — App Router, серверные и клиентские компоненты
* **Fabric.js 6** — canvas-редактор
* **SQLite** через **better-sqlite3** — база данных
* **Resend** — отправка email-верификации
* **TypeScript**

---

## Возможности

### Инструменты редактора

| Клавиша | Инструмент                           |
| ------- | ------------------------------------ |
| `V`     | Выбор — перемещение, ресайз, поворот |
| `H`     | Рука — панорамирование холста        |
| `P`     | Карандаш — свободное рисование       |
| `R`     | Прямоугольник                        |
| `O`     | Эллипс                               |
| `T`     | Треугольник                          |
| `D`     | Ромб                                 |
| `L`     | Линия                                |
| `A`     | Стрелка с наконечником               |
| `X`     | Текст — inline-редактирование        |
| `N`     | Стикер                               |
| `E`     | Ластик                               |
| `I`     | Изображение — загрузка с компьютера  |

### Стикеры

* Новый инструмент **Sticker** вместо старого `sticky`.
* Горячая клавиша изменена с `S` на `N`.
* Стикер создаётся как группа из `Rect + Textbox`, а не как один `Textbox` с кастомным фоном.
* Размер по умолчанию — **200×200**.
* Есть отдельная палитра из 8 цветов для стикеров.
* Цвет стикера можно менять в правой панели свойств.
* При выборе стикера скрываются обычные настройки контура и стандартной заливки.
* Двойной клик по стикеру открывает редактирование текста.
* Текст автоматически уменьшается, чтобы помещаться внутри стикера.
* Длинные строки разбиваются по символам, чтобы текст не выходил за границы.
* После изменения размера стикера масштаб “запекается” в реальные размеры, поэтому текст не растягивается.
* Стикеры сохраняются и восстанавливаются через `canvasState`.

### Холст

* **Undo/Redo** — `Ctrl+Z` / `Ctrl+Y`, история до 60 состояний.
* **Зум** — колесо мыши, `Ctrl++`, `Ctrl+-`, диапазон 5%–2000%.
* **Fit to screen** — `Ctrl+0`.
* **Фон** — без фона / точки / сетка.
* Фон синхронизируется с зумом и панорамированием.
* Стиль фона сохраняется в базе данных.
* **Автосохранение** каждые 30 секунд.
* **Экспорт PNG** в 2× качестве.
* Поддерживается `strokeUniform`, чтобы толщина обводки не растягивалась при масштабировании.
* Панорамирование средней кнопкой мыши работает с любым инструментом.

### Совместная работа

* Доской можно поделиться по email.
* Владелец выбирает роль: **Редактор** или **Зритель**.
* **Редактор** может рисовать, редактировать и сохранять доску.
* **Зритель** видит доску в режиме «Только просмотр», может панорамировать и масштабировать.
* Роль можно изменить через бейдж роли пользователя.
* Изменения другого пользователя подгружаются автоматически через polling каждые 3 секунды.
* Только владелец может переименовывать, удалять доску и управлять доступом.
* Пользователь, которому выдали доступ, может убрать доску у себя, не удаляя её для остальных.
* На дашборде шареные доски помечаются бейджем **«ПОДЕЛЕНО»**.

### Дашборд

* Список собственных и шареных досок.
* Сортировка по дате обновления.
* Превью-миниатюры досок.
* Создание новой доски.
* Переименование доски.
* Удаление своей доски.
* Удаление шареной доски только из своего списка.

### Аутентификация

* Регистрация с email-верификацией через Resend.
* Вход и выход.
* Восстановление пароля.
* Сброс пароля по токену.
* HTTP-only cookie-сессии на 30 дней.
* Защита маршрутов через middleware.

## База данных

SQLite база создаётся автоматически при первом запуске.

Файлы базы:

```text
whiteboard.db
whiteboard.db-shm
whiteboard.db-wal
```

Они добавлены в `.gitignore` и не должны попадать в репозиторий.

### Таблица `users`

| Поле                | Тип         | Описание                   |
| ------------------- | ----------- | -------------------------- |
| `id`                | INTEGER PK  | ID пользователя            |
| `name`              | TEXT        | Имя пользователя           |
| `email`             | TEXT UNIQUE | Email                      |
| `password`          | TEXT        | Хешированный пароль        |
| `emailVerified`     | INTEGER     | `0` или `1`                |
| `verificationToken` | TEXT        | Токен подтверждения email  |
| `tokenExpiry`       | DATETIME    | Срок действия токена       |
| `resetToken`        | TEXT        | Токен сброса пароля        |
| `resetTokenExpiry`  | DATETIME    | Срок действия reset-токена |
| `created_at`        | DATETIME    | Дата регистрации           |

### Таблица `sessions`

| Поле         | Тип         | Описание      |
| ------------ | ----------- | ------------- |
| `id`         | INTEGER PK  | ID сессии     |
| `userId`     | INTEGER FK  | Пользователь  |
| `token`      | TEXT UNIQUE | Токен сессии  |
| `expiresAt`  | DATETIME    | Срок действия |
| `created_at` | DATETIME    | Дата создания |

### Таблица `boards`

| Поле          | Тип        | Описание                  |
| ------------- | ---------- | ------------------------- |
| `id`          | INTEGER PK | ID доски                  |
| `userId`      | INTEGER FK | Владелец                  |
| `name`        | TEXT       | Название доски            |
| `canvasState` | TEXT       | JSON-состояние холста     |
| `thumbnail`   | TEXT       | Base64-превью             |
| `bgStyle`     | TEXT       | `none`, `grid` или `dots` |
| `updated_by`  | INTEGER FK | Кто последним сохранил    |
| `created_at`  | DATETIME   | Дата создания             |
| `updated_at`  | DATETIME   | Дата обновления           |

### Таблица `board_shares`

| Поле         | Тип        | Описание                |
| ------------ | ---------- | ----------------------- |
| `id`         | INTEGER PK | ID записи               |
| `boardId`    | INTEGER FK | Доска                   |
| `userId`     | INTEGER FK | Пользователь с доступом |
| `role`       | TEXT       | `editor` или `viewer`   |
| `created_at` | DATETIME   | Дата выдачи доступа     |

`UNIQUE(boardId, userId)` не позволяет добавить одного пользователя к одной доске несколько раз.

## Запуск проекта

### 1. Установить зависимости

```bash
npm install
```

### 2. Создать `.env.local`

```bash
cp .env.example .env.local
```

Для PowerShell:

```powershell
cp .env.example .env.local
```

### 3. Заполнить переменные окружения

```env
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

`RESEND_API_KEY` нужен для отправки писем подтверждения email и восстановления пароля.

### 4. Запустить проект

```bash
npm run dev
```

После запуска открыть:

```text
http://localhost:3000
```

## API

### Доски

* `GET /api/boards` — получить список своих и шареных досок.
* `POST /api/boards` — создать новую доску.
* `GET /api/boards/[id]` — получить доску.
* `PUT /api/boards/[id]` — обновить доску.
* `DELETE /api/boards/[id]` — удалить свою доску или убрать шареную доску из своего списка.
* `GET /api/boards/[id]/sync` — получить `updated_at` и `updated_by` для синхронизации.

### Шаринг

* `GET /api/boards/[id]/share` — получить список пользователей с доступом.
* `POST /api/boards/[id]/share` — выдать доступ по email.
* `PATCH /api/boards/[id]/share` — изменить роль пользователя.
* `DELETE /api/boards/[id]/share?userId=N` — забрать доступ.

### Аутентификация

* `POST /api/user/register` — регистрация.
* `GET /api/auth/verify-email?token=...` — подтверждение email.
* `POST /api/auth/resend-verification` — повторная отправка письма подтверждения.
* `POST /api/auth/login` — вход.
* `POST /api/auth/logout` — выход.
* `GET /api/auth/me` — текущий пользователь.
* `POST /api/auth/forgot-password` — запрос сброса пароля.
* `POST /api/auth/reset-password` — установка нового пароля.
